# Content R0 — 用流行病学算文案的传播力

> 把疫情模型搬到内容传播上：每条内容都有一个"基本再生数 R0"。
> R0 > 1 的内容会扩散，R0 < 1 的会消失。
> 本项目用本地大模型扮演一群有人格设定的"虚拟读者"来估计这个数，
> 并基于这个反馈反过来迭代写出更易传播的内容。

---

## 一句话能解释吗？

写了一条想发的微博 / 朋友圈 / 公众号开头，
你大概知道它"还行"，但不知道它有没有传播力。

这个项目让 25 个虚拟读者（不同年龄 / 职业 / 兴趣 / 性格）
逐个看你的内容，每个人独立回答：
**"会不会转？转给谁？为什么？"**

把这些回答聚合起来，就得到一个数字 —— **R0（基本再生数）**：

> 平均一个看到这条内容的人，会让多少新的人也看到。

- R0 ≈ 1.5：稳定扩散，能涨粉
- R0 ≈ 0.6：传几波就熄火
- R0 ≈ 3：病毒级（但通常意味着内容很"重")

更进一步：你给程序一段**创作要求**（比如"为新产品 X 写一条朋友圈"），
它会自己生成草稿，对着 R0 反馈不断改写，
最后产出**既切题、又能被转发**的内容。

---

## 它能做什么 / 不能做什么

✅ **能**：
- 比较两条文案哪条更易扩散，给出数字依据
- 告诉你转发者是哪类人（年龄段 / 职业 / 平台）
- 给出"不转发的人为什么不转"的具体理由
- 基于反馈自动迭代出高传播力内容

❌ **不能**：
- 给出能直接对应真实平台数据的绝对 R0 值（这是模型估计，不是观测）
- 替代真实 A/B 测试
- 保证生成的内容是"善意"的——真实平台上的高 R0 内容往往就是
  愤怒、恐惧、对立、猎奇的内容，这是算法生态的事实

## 快速开始

### 0. 环境

需要：

- 一张能跑大模型的 GPU（项目用的是 RTX 3090，跑 35B Q4_K_M）
- Python 3.10+
- `llama-cpp-python`（带 CUDA 编译，参考 [官方文档](https://github.com/abetlen/llama-cpp-python)）

模型路径在 `config.py` 里改，默认是：

```
/home/zhou/shared/model/Qwen3.5-35B-A3B-Uncensored-HauhauCS-Aggressive-Q4_K_M.gguf
```

> 这个模型加载后基本占满显存，所以本程序运行时建议独占 GPU。

### 1. 初始化虚拟人口（只要做一次）

```bash
python main.py init-personas --n-personas 25
```

这会生成 25 个有完整设定的虚拟读者并存到 `data/personas.json`。
随机种子固定 (=42)，所以你和别人跑同样的内容，面对的是**同一群人**，
R0 数值之间可比较。

要看长什么样：

```bash
python main.py init-personas --n-personas 5
```

会打出几个示例，类似：

```
P000:
  - 年龄段：中老年(56-65)
  - 职业：退休人员
  - 所在地类型：三四线城市
  - 学历：高中/中专
  - 主要兴趣：健身养生、星座命理、传统文化
  - 性格：保守谨慎、怕被骗
  - 最常使用的平台：微信朋友圈+公众号
```

### 2. 评估一条内容

```bash
python main.py evaluate --content "你的内容..." --out result.json
```

输出每个虚拟读者的判断、最终 R0、以及哪类人在转。

### 3. ★ 主推：从 brief 生成内容

把你想写什么写到 `word.txt` 里。例如：

```
为我们公司新出的咖啡店写一条小红书种草文案，
店在静安寺，特色是只用单一产地豆子和手冲，
希望吸引 25-35 岁的咖啡爱好者，
不要太硬广，要有点"懂咖啡的人才懂"的优越感。
```

然后跑：

```bash
python main.py generate --out result.json --best-out best.txt
```

程序会：

1. 读取 `word.txt` 作为创作要求
2. 用 LLM 生成 5 条不同风格的草稿（gen 0）
3. 对每条算两个分：
   - **R0**：传播力（25 个虚拟读者投票）
   - **切题分**：0-10，由 LLM 当裁判判断是否符合要求
   - 综合 fitness = R0 × 切题分/10
4. 保留 top-3，**把"读者真实反馈"喂给 LLM**，让它针对性改写出新一代变体
5. 重复 3 代，输出最佳成品到 `best.txt`

每一代你能看到具体反馈：

```
转发者的真实理由：
  - 32岁互联网工程师/愤世嫉俗：这话太对了，转给同事看
  - 45岁家庭主妇/情绪化：心疼这小哥，得让大家看看

不转发者的真实理由：
  - 60岁退休/保守谨慎：看不太懂年轻人这些
  - 23岁学生/理性冷静：感觉是煽情套路，不想发
```

下一代变体就是带着这些反馈写出来的。

### 4. 自由进化（不限制主题）

如果你只是想看一条内容怎么改更易传播（不要求保留原意）：

```bash
python main.py evolve --seed "你的种子内容..." \
    --generations 3 --variants 3 --top-k 3 --out history.json
```

### 5. 看几个例子（demo）

```bash
python demo.py
```

跑 5 条手工设计的对比内容（技术干货 / 社会情绪 / 养生谣言 / 平淡日常 / 争议观点），
快速验证程序工作正常。预期看到的相对排序：

> 社会情绪 > 争议观点 > 养生谣言 > 技术干货 > 平淡日常

## 它是怎么算 R0 的？

简化为一行公式：

```
R0(c) = E_{p 是转发者} [ κ_p(c) × β_p(c) ]
```

其中：

- **κ_p(c)**：人物 p 转发后预计触达多少人（直接问 LLM "你的社交圈有多大"）
- **β_p(c)**：被 p 触达的人群中，再次转发的平均概率（在虚拟人口里查 p 指定的"目标受众"，取他们的转发概率均值）

直觉：传播 = "有人愿意转 (κ × β 里的 β)" + "转的人能触达不少人 (κ)" + "下一批人也会继续转 (β)"。

这是**一阶解析估计**，不是真的跑多代分支模拟（多代模拟需要 O(K^G) 次 LLM 调用，35B 模型扛不住）。
**绝对数值不准，相对排序可信** —— 而对优化（找哪条更易传播）来说，相对排序就够了。

## 主要参数（在 `config.py`）

| 参数 | 默认值 | 说明 |
|---|---|---|
| `DEFAULT_POPULATION_SIZE` | 25 | 虚拟读者数。50+ 才让 R0 数值稳定 |
| `PERSONA_TEMPERATURE` | 0.85 | 太低 → 所有人答案趋同，区分度差 |
| `VARIATION_TEMPERATURE` | 0.95 | 写变体的温度，故意高，鼓励探索 |
| `ADHERENCE_TEMPERATURE` | 0.3 | 切题判断要稳定，温度低 |
| `GENERATE_MIN_ADHERENCE` | 5 | 切题分 < 5 直接淘汰，不算 R0 省钱 |

`generate` 命令的代数 / 变体数 / top-k 这些都可以在命令行覆盖。

## 跑一次大概多久？

在 RTX 3090 + Qwen3.5-35B-A3B Q4_K_M 上：

| 操作 | 时间 |
|---|---|
| 单次 persona 查询 | 1-3 秒 |
| 评估一条内容（25 persona） | 0.5-1.5 分钟 |
| `generate` 默认配置（3 代 × ~12 候选） | 15-40 分钟 |
| `demo.py`（5 条对比） | 3-7 分钟 |

加了磁盘缓存：同一 (persona, content) 对永远只调用一次 LLM。
精英在下一代继续被评估时直接命中缓存。

## 局限和真话

**1. LLM 模拟人 ≠ 真人。**
Qwen 扮演的"中老年阿姨"和真实的中老年阿姨在判断转不转某条内容时可能差很远。
**绝对 R0 数值没有校准价值，相对排序才有。**
要校准，需要拿真实平台数据回归一下。

**2. 受众匹配是字符串子串。**
转发者说"我会转给育儿妈妈"，程序就找标签里包含"育儿"或"妈妈"的虚拟读者。
简单粗暴，可能错配。生产用应该改成 embedding 匹配。

**3. 一阶近似，不是多代模拟。**
平台算法分发偏好、舆论反扑、话题竞争，都没建模。

**4. LLM 自洽性偏差。**
当同一个 LLM 既扮演读者、又生成内容时，进化容易陷入这个 LLM 自己的偏好
（"我觉得好的我也觉得别人觉得好"）。要破这个，需要至少两个不同模型分担读者和创作者的角色。

**5. 价值观风险（最重要）。**
系统会学到"愤怒 / 对立 / 恐惧 / 猎奇"内容 R0 更高 —— 这正是真实平台算法的内核。
如果你拿生成结果去发布，你就在亲手生产带情绪杠杆的内容。

> **作者立场：**
> 这套东西最适合用于研究和防御 ——
> 帮你识别哪类内容容易病毒式传播，从而知道在内容生态中**该警惕什么**、
> **该如何应对**，或者**自己的内容到底为什么传不出去**。
>
> 把它用作"造爆款机器"在技术上完全可行，但不要用它来制造谣言、
> 操纵情绪、煽动对立。这是常识，不是法律免责。

**6. 没有内容安全护栏。**
程序底层用的是 Uncensored 版本的模型，没有内置毒性 / 事实性过滤。
如果你要把它扩展成可发布的生成器，请自己加一层过滤。

## 可以怎么扩展

- **真实数据校准**：抓一批微博 / 小红书的转发计数，拟合一个 真实R0 ↔ 模拟R0 的回归
- **多代精确模拟**：用更小的本地模型（7B 量级）做下游 persona，跑真正的分支过程
- **图结构人口**：把 persona 之间的"audience 标签匹配"换成显式社交图 + IC/LT 扩散模型
- **疲劳维度**：`fatigue_days` 已经收集了，可以加时间衰减项研究"同主题连推 N 天 R0 怎么衰减"
- **多模态**：把 content 扩展为带图 / 视频描述的复合输入
- **双模型解耦**：换两个不同来源的模型分担读者 / 创作者，破解 LLM 自洽偏差
